# NFT SKILL distribution package pattern

Use this when the user wants to share their Web3/NFT Hermes skill setup with friends or a Telegram group.

## Package title and shape

- Title the shareable artifact **NFT SKILL**.
- Produce a ZIP package, not only prose.
- Include a top-level `README.md`, `docs/TUTORIAL_INSTALL.md`, `install/install_nft_skill.sh`, `cron-templates/`, `radar-scripts/`, `skills/`, and `meta/manifest.txt`.
- Make the tutorial friend/group-ready: concise Indonesian, install steps, Telegram topic routing examples, cron examples, anti-spam rules, and troubleshooting.

## Curated related skills to include

Prefer these Web3/NFT-supporting skills when present:

| Skill | Why include |
|---|---|
| `nft-auto-mint` | NFT contract research, mint simulation/execution, Waypoint/X radar references. |
| `hermes-crypto-agent` | Multi-chain crypto/NFT/wallet/monitoring workflows. |
| `web3-ops` | Batch Web3 operations, RPC fallback, mass wallet/mint/airdrop ops. |
| `universal-minter` | General NFT mint workflows across OpenSea/Manifold/Zora/SeaDrop. |
| `nft-wl-filler` | WL/allowlist form and X task workflows. |
| `batch-ops` | Parallel/resumable batch execution. |
| `xurl` | X/Twitter search/post/API support for NFT radar. |
| `polymarket` | Crypto market info. |
| `blogwatcher` | RSS/blog/project monitoring. |
| `hermes-agent` | Hermes setup, Telegram gateway, cron, skills/tools. Include as reference only; do not patch protected bundled/hub skills. |

## Radar scripts to include

Include only source/scripts, not env/state/credentials:

- `x_nft_mint_radar.py`
- `x_nft_mint_radar_format.py`
- `x_nft_mint_radar_general.sh`
- `x_nft_mint_radar_wl.sh`
- `waypoint_hot_mint_radar.py`
- `waypoint_hot_mint_radar.sh`

## Cron template defaults

Create sanitized templates with placeholder chat IDs:

- Free mint / Mint Live / Stealth mint NFT → `telegram:-100XXXXXXXXXX:37039`, script `x_nft_mint_radar_general.sh`, `no_agent=true`, every 10–20m.
- WL / Allowlist / Whitelist → `telegram:-100XXXXXXXXXX:53949`, script `x_nft_mint_radar_wl.sh`, `no_agent=true`, every 10–30m.
- Waypoint HOT NFT Live Mint → `telegram:-100XXXXXXXXXX:54012`, script `waypoint_hot_mint_radar.sh`, `no_agent=true`, every 10m.

## Security exclusions

Never include:

- `.env`, `*.env`, `x-radar.env`
- Telegram bot token
- X `auth_token` or `ct0`
- API/RPC keys
- private keys, seed phrases, mnemonic files
- `vault.enc`
- personal wallet address reference files like `wallet-addresses.md`
- `auth.json`, cookie files, state DBs, session logs
- `__pycache__`, backups, generated bytecode

Verify ZIP before sending:

- `zipfile.testzip() is None`
- suspect path count is zero for secret-like path names
- content scan has no Telegram bot-token pattern and no actual `auth_token=...`, `ct0=...`, `private_key=...`, `mnemonic=...`, or seed assignments
- top-level README and tutorial exist

## Installer script pattern

The installer should copy package skills into `$HERMES_HOME/skills/` and radar scripts into `$HERMES_HOME/scripts/`, then chmod scripts executable. It should not write credentials. Tell users to configure their own Telegram/X/RPC secrets locally.
